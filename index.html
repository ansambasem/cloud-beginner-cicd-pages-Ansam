What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. Continuous Integration means that when developers push code, the system automatically checks it. Continuous Deployment means that the application is automatically updated after changes are approved. This process helps reduce errors and makes development faster and more organized. It is widely used in cloud systems to automate software delivery.

My Pipeline

When I push my code to the main branch on GitHub, a workflow starts automatically. It checks that all required files exist in the repository. If everything is correct, the website is deployed to GitHub Pages. This allows the site to update automatically after every push.
